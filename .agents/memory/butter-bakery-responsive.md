## Navbar breakpoint
- Desktop nav (all items + "ABOUT US" button) shows at `lg:` (1024px+).
- Hamburger menu shows at `lg:hidden` — so 768-1023px (tablet) gets the mobile overlay.
- **Why:** At 768px the 5 nav items + language toggle + "ABOUT US" button overflow and "ABOUT US" wraps to two lines. Moving to `lg:` fixes this cleanly.

## Hero overlay
- Mobile (< 640px): `from-black/70 via-black/50 to-black/80` — darker for text readability over busy background photo.
- sm+ (640px+): `sm:from-black/60 sm:via-black/30 sm:to-black/70` — lighter to show the beautiful restaurant interior.
- H1 uses `[text-shadow:0_2px_24px_rgba(0,0,0,0.7)]` for extra legibility over any background.

## CSS utilities
- `scrollbar-hide` is NOT generated by Tailwind v4 — must be manually defined in `index.css`.
- `font-arabic` class manually defined in index.css (maps to Noto Kufi Arabic).

## News page
- NewsItem interface has `imagePosition?: string` for controlling `object-position` on featured card images.
- Seven signing photo (id: 0) uses `imagePosition: "center bottom"` to show people's faces over the TV/branding area.

## Footer nav
- Footer navLinks support both hash anchors (`#story`) and page routes (`/news`).
- Footer.tsx uses `<Link>` (wouter) for hrefs starting with `/`, and `<a>` for hash anchors.
- News link added to both EN and AR footer navLinks.

## Gallery / FollowUs
- Gallery: `auto-rows-[120px] sm:auto-rows-[140px] md:auto-rows-[185px]` for good cell proportions across devices.
- FollowUs: `grid-cols-3 sm:grid-cols-4 lg:grid-cols-6` — avoids cramped 3-col on all devices.

## Branches
- Secondary image strip: `h-24 md:h-36` (was h-20 md:h-28) — taller for better visual weight.
